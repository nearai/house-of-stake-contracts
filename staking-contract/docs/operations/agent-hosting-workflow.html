NEAR Staking For Agent Hosting

End-to-end workflow for the chat-api subscription stack: users stake NEAR through the staking contract, the contract delegates through the NEAR AI validator pool, and the server turns verified subscription state into hosted-agent entitlements and stake-based monthly credits.

Client: app, wallet, user Server: chat-api config and policy Staking Contract: catalog and state NEAR AI Validator: staking pool
Client user surface

1Select agent plan

User opens billing, selects Starter, Basic, or Pro, and connects a NEAR wallet account.

4Sign NEAR transaction

Wallet signs storage_deposit when needed, then lock or update_subscription with the selected on-chain price_id and staked NEAR amount.

9Use hosted agents

Client receives the active entitlement, available agent count, and stake-based monthly credits from the server.

Server chat-api

2Load runtime config

Reads subscription_plans, HOS provider price_id values, agent limits, and credits_per_staked_near_nano_usd.

3Prepare quote

Combines server policy with staking-contract catalog views so the client sees the correct min/max NEAR range and plan metadata.

8Authorize service

Queries subscription or farm views for the account, checks active status and staked NEAR, then computes hosted-agent limits and monthly credits.

Staking Contract testnet/mainnet

2Catalog source of truth

Stores products, recurring prices, min stake in amount, max stake in metadata.max_amount, and subscription state.

5Validate and settle

Checks account storage, active price/product, stake range, and validator status. Runs the shared epoch pipeline before minting or updating user subscription state.

7Expose state

Emits events and exposes views such as get_subscription_for_product, get_subscriptions_for_account, and pending-unstake views.

NEAR AI Validator staking pool

AOwnership authority

Catalog admin mutations verify validator ownership through pool get_owner_id, so the validator owner controls its product catalog.

6Pool operations

Contract drives get_account, deposit_and_stake, unstake, and withdraw with one in-flight settlement pipeline per pool.

BEconomic backing

Staked NEAR remains delegated to the validator pool while the contract tracks user shares, exits, and claimable withdrawals.

Subscribe

User chooses a plan and stakes enough NEAR for the on-chain price tier.

Entitle

chat-api maps verified staking state to agent limits and monthly credits.

Renew Or Change

Renewals call lock; upgrades and downgrades call update_subscription.

Exit

Cancelled or reduced stake queues unstake; users later withdraw claimable NEAR after validator settlement.

Component Writes Reads
Client Signed NEAR transactions: storage registration, subscribe, update, cancel, withdraw. Plan metadata, wallet state, subscription state, agent entitlement returned by server.
Server chat-api system config and off-chain agent provisioning state. Staking contract products, prices, subscriptions, farm/account views, and NEAR RPC receipts.
Staking Contract Catalog, subscriptions, locks, farm positions, pending unstake and claim buckets. Validator pool owner, validator account balance, current epoch, attached deposits.
NEAR AI Validator Pool stake, unstake, and withdraw accounting through standard staking-pool methods. Contract calls from the staking contract; owner identity for catalog authorization.
